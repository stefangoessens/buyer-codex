# Hosman

Role in hierarchy: primary structural reference.

## What Was Captured

- Hero estimator
- Immediate proof strip
- Property-card rail
- "Why choose us" proof block
- Three-step conversion section
- Mobile stacked hero and card rail

## Key Observations

### Structure

- Hosman makes the user act immediately: heading, estimator, friction-reducer row.
- Social proof appears before broader feature explanation.
- Listings appear surprisingly early, which makes the whole page feel grounded in real inventory instead of abstract marketing.
- The three-step selling method is an ideal reference for buyer-codex process storytelling.

### Style notes that matter

- Headings: `Poppins, Verdana, sans-serif`
- Input/body utility: `Rubik, sans-serif`
- CTA blue: `rgb(80, 118, 246)`
- Heading ink: `rgb(33, 23, 90)`
- Body ink: `rgb(30, 26, 55)`
- Page background: `rgb(252, 251, 255)`
- Property card radius: `24px`
- Property card shadow: `0 0 15px 0 rgba(0, 0, 0, 0.07)`

### Product implications

- Buyer-codex homepage, calculator, and intake should all start from the same estimator-led pattern.
- Property proof belongs near the top of the funnel, not buried far below generic features.
- Repeated CTA copy should stay focused on estimating or starting, not proliferate into many similar actions.

## Adopt

- Hero estimator pattern
- Proof strip directly below the hero
- Early property rail
- Numbered three-step conversion explanation
- Repeated but disciplined CTA placement

## Reject

- Deep footer taxonomy
- Overly verbose FAQ body copy
- Too many near-duplicate CTA phrasings
